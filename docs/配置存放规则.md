# 配置存放规则

本文档约定软件配置如何进入仓库，重点是可恢复、可审阅和不泄露敏感信息。

## 放置位置

- 单文件配置直接放到对应软件目录，例如 `apps/git/config.gitconfig`。
- 多文件配置放到 `config/` 子目录，例如 `apps/wezterm/config/`、`apps/neovim/config/`。
- 插件或扩展列表使用 `extensions.md`。
- 命令导出的环境或配置可使用原格式，例如 `go-env.json`、`rustup-show.md`。

## 脱敏规则

- 用户目录统一替换为 `%USERPROFILE%`。
- 邮箱统一替换为 `<email>`。
- token、password、secret、api key、cookie、session 等字段统一替换为 `<redacted>`。
- 代理订阅、远程连接凭据、证书、私钥、授权文件不入库。

## 跳过规则

- `.env`、`.npmrc` 中疑似认证项、`hosts.yml`、`.docker/config.json` 等高风险文件默认跳过。
- 浏览器 Profile、聊天记录、云盘同步内容、密码库、数据库数据目录不入库。
- 二进制缓存、字体文件、截图、历史记录和构建产物不作为配置保存。

## 导出脚本

- 使用 `scripts/export-configs.ps1` 导出常见配置。
- 脚本只读取白名单路径，并在写入仓库前做通用脱敏。
- 导出后检查 `apps/config-export-summary.md` 了解本次导出的配置范围。

## 人工复核

- 新增配置前先确认是否包含账号、路径、访问地址或私有项目名。
- 无法确认是否敏感时，优先写入说明文件，不保存原始配置。
- 提交前建议搜索 `<redacted>` 以确认敏感字段已经被替换。
