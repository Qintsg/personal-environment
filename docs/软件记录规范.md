# 软件记录规范

每个软件目录都应有一个 `install.md`，用于说明默认安装平台、当前版本、安装来源和恢复命令。

## install.md 必填内容

- 软件名称：作为一级标题。
- 默认平台：必须写明默认安装平台为 Windows。
- 当前记录：记录版本、来源和包 ID；没有包 ID 时写“待补充”。
- 安装方式：记录可执行的安装命令，或说明需要手动安装。
- 备注：用于补充配置路径、插件清单或迁移注意事项。

## 安装命令约定

Windows PowerShell 为默认命令环境。

```powershell
winget install --id <Package.Id> --exact --accept-package-agreements --accept-source-agreements --disable-interactivity
scoop install <package>
choco install <package> -y
npm install -g <package>
cargo install <crate>
dotnet tool install --global <package>
```

## 后续扩展

- 插件较多的软件可新增 `extensions.md`。
- 配置较多的软件可新增 `config.md` 或直接保存配置文件。
- 安装步骤复杂的软件可在 `install.md` 中增加“安装后检查”和“常见问题”。

## 自动同步

- 使用 `scripts/export-packages.ps1` 导出当前机器的软件包信息。
- `winget export`、注册表、AppX、pip、pnpm、yarn、Go binary 和 PowerShell module 结果保存在 `apps/_inventory/`，用于完整复核。
- Scoop、npm 全局包、cargo install、dotnet tool 等可恢复来源会自动补齐或更新对应 `install.md`。
- 新的 winget 软件目录默认不自动创建；如需把所有 winget 可导入包展开成目录，可显式传入 `-AddNewWingetDocs` 后再人工复核运行库和系统组件。
