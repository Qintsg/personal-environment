# 程序存放规则

本文档约定仓库中各程序相关内容的存放方式，方便后续查找、迁移和维护。

## 基本规则

- 每个软件使用一个独立目录，统一放在 `apps/` 下。
- 软件目录名使用小写英文、数字和连字符，例如 `visual-studio-code`、`docker-desktop`。
- 每个软件目录至少包含 `install.md`，用于记录安装方式。
- Windows 是默认安装平台；其他系统的安装方式可以继续补充到同一个 `install.md` 中。
- 不提交 `.env`、token、私钥、账号密码、授权文件等敏感内容。

## 推荐结构

```text
apps/
  visual-studio-code/
    install.md
    settings.json
    extensions.md
  git/
    install.md
    config.gitconfig
  docker-desktop/
    install.md
    credentials-skipped.md

docs/
  程序存放规则.md
  软件记录规范.md
  配置存放规则.md
```

## 记录优先级

- 优先记录可复现的包管理器命令，例如 `winget`、`scoop`、`choco`、`npm`、`cargo`、`dotnet tool`。
- 无法确认包管理器来源的软件，先记录为手动安装，并说明后续需要补充来源。
- 同一软件有多个安装来源时，以当前实际使用来源为准。
- 驱动、运行库和系统组件只在确实需要恢复环境时记录。

## 命名建议

- 包管理器工具使用原包名或接近包名的目录名。
- npm 全局包使用 `npm-包名`，作用域包去掉 `@` 并用连字符连接。
- Cargo 工具使用 crate 名称。
- 中文软件使用常见英文名或拼音目录名，并在 `install.md` 标题中保留中文原名。
